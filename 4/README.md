## Как реализован бот

Бот написан на **Python 3** и построен на **гибридной архитектуре**, объединяющей три подхода: **регулярные выражения** (модуль `re`), **обработку естественного языка через spaCy** и **конечный автомат (Finite State Machine)** для управления диалогом.

### 🔹 Управление состоянием диалога (DialogState)

Введён класс `DialogState` с двумя состояниями:
- `START` — обычный режим ожидания команд
- `WAIT_CITY` — режим ожидания названия города после того, как пользователь спросил о погоде, но не указал город

Состояние сохраняется в базу данных (`save_state()` / `load_state()`), поэтому при перезапуске бота диалог может продолжиться с того же места (хотя в текущей версии загрузка состояния не используется в `__init__`).

### 🔹 Обработка сообщений (`bot.py`)

Метод `process()` работает по следующему алгоритму:

1. **Проверка состояния** — если бот находится в `WAIT_CITY`, любое сообщение пользователя интерпретируется как название города. Состояние сбрасывается в `START`, вызывается `handlers.get_weather()` с текстом сообщения в качестве города.

2. **Проверка паттернов** — последовательный перебор регулярных выражений из `patterns.py`:
   - Приветствия → `handle_greeting()`
   - Сложение чисел → `handle_addition()`
   - Запоминание имени → `set_name()`

3. **Проверка триггеров погоды** — если в сообщении есть слова `погод`, `температур`, `градус` или `прогноз`:
   - Через `extract_city()` пытается извлечь город из сообщения
   - Если город найден — сразу возвращает погоду
   - Если не найден — переводит бота в состояние `WAIT_CITY` и возвращает вопрос «В каком городе вас интересует погода?»

4. **Ответ по умолчанию** — если ни одно условие не сработало, возвращается «Я не понимаю запрос».

### 🔹 Извлечение города (spaCy)

Метод `extract_city()` использует русскую модель **`ru_core_news_sm`**:
- Сначала ищет именованные сущности типов **GPE** (геополитические единицы: страны, города) и **LOC** (локации). При нахождении возвращает лемму (нормальную форму) слова.
- Если сущности не найдены — текст разбивается на слова, ищется слово, не входящее в стоп-список (`я`, `привет`, `погода`, `какая` и др.), очищается от пунктуации и возвращается с заглавной буквы.
- Если ничего не найдено — возвращает `None`, что вызывает переход в состояние ожидания города.

### 🔹 Паттерны (`patterns.py`)

Список регулярных выражений для точных команд:
- `^(привет|здравствуйте|добрый день)` — приветствие
- `^(пока|до свидания)` — сигнал выхода (хотя выход теперь обрабатывается отдельно в основном цикле)
- `(\d+)\s*\+\s*(\d+)` — сложение двух чисел
- `меня зовут ([а-яА-Яa-zA-Z]+)` — сохранение имени пользователя

### 🔹 Обработчики (`handlers.py`)

- **`get_weather(city)`** — отправляет GET-запрос к **Weatherstack API** через `requests`. Извлекает температуру, скорость ветра и описание погоды. Описание переводится на русский через словарь `WEATHER_TRANSLATIONS`. При ошибке подключения или отсутствии города возвращает соответствующее сообщение.
- **`handle_addition()`** — извлекает два числа из совпадения, преобразует в `float`, складывает.
- **`handle_greeting()`** — возвращает приветственную строку.

### 🔹 Логирование и хранение (`logger.py`)

Используется **SQLite** с тремя таблицами/функционалом:

- **Таблица `logs`** — хранит полную историю диалогов: временная метка, текст пользователя, ответ бота и **состояние бота** на момент ответа. Это позволяет анализировать, в какой фазе диалога были заданы вопросы.
- **Таблица `user_info`** — ключ-значение для хранения:
  - `user_name` — имя пользователя
  - `dialog_state` — текущее состояние диалога (для сохранения между сессиями)

Функции:
- `log_message(user_input, response, state)` — запись диалога с состоянием
- `save_user_name()` / `load_user_name()` — работа с именем
- `save_state()` / `load_state()` — сохранение и загрузка состояния (функция `load_state()` определена, но пока не используется в `__init__`)

### 🔹 Основной цикл

При запуске загружается сохранённое имя пользователя. Бот входит в бесконечный цикл:
- Если пользователь вводит `пока` или `выход` — цикл прерывается с прощанием
- Иначе сообщение передаётся в `bot.process()`
- Ответ выводится на экран с временной меткой
- Диалог логируется в БД вместе с текущим состоянием бота

### 🔹 Ключевые особенности

1. **Конечный автомат** — бот может находиться в режиме ожидания города, что позволяет поддерживать естественный диалог: пользователь пишет «Какая погода?», бот переспрашивает «В каком городе?», и следующий ответ пользователя автоматически интерпретируется как город.

2. **Гибкое извлечение города** — благодаря spaCy бот понимает не только «погода в Москве», но и «какая температура в Санкт-Петербурге?», «прогноз на завтра в Казани».

3. **Сохранение состояния** — состояние диалога сохраняется в БД, что теоретически позволяет восстанавливать диалог после перезапуска (хотя в текущей версии загрузка не реализована в `__init__`).

4. **Приоритет паттернов** — регулярные выражения проверяются до триггеров погоды, поэтому фраза «привет + 5» сначала распознается как приветствие, а не как попытка сложения.
