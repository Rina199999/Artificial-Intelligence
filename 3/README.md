## Как реализован бот

Бот написан на **Python 3** и использует комбинированный подход: **регулярные выражения** (модуль `re`) для точных команд и **библиотеку spaCy** для извлечения смысла из естественного языка.

### 🔹 Обработка сообщений (`bot.py`)

Класс `ChatBot` получает сообщение, приводит его к нижнему регистру и проверяет наличие **триггеров погоды**: слова `погод`, `температур`, `градус`, `прогноз`. Если триггер найден — бот пытается извлечь название города из фразы. Если город успешно извлечён — вызывается `handlers.get_weather(city)`, иначе погода запрашивается для города по умолчанию (Нижний Новгород). Если ни один триггер не сработал — возвращается ответ «Я не понимаю запрос».

### 🔹 Извлечение города с помощью spaCy

Метод `extract_city()` использует предобученную русскую модель **`ru_core_news_sm`**:
- Сначала пытается найти сущности типов **GPE** (геополитическая единица, например, страна/город) и **LOC** (локация). При нахождении возвращает лемму (нормальную форму) слова.
- Если сущности не найдены — разбивает текст на слова и ищет слово, которое не входит в стоп-список (`я`, `привет`, `погода`, `в` и т.д.), очищает его от знаков пунктуации и возвращает с заглавной буквы.
- Если ничего не найдено — возвращает `None`, и тогда используется город по умолчанию.

### 🔹 Паттерны (`patterns.py`)

Файл возвращает список кортежей с регулярными выражениями и обработчиками, **но в текущей версии они фактически не используются** — основной цикл `process()` больше не вызывает `self.patterns`. Паттерны остались в коде для обратной совместимости:
- Приветствия → `handle_greeting()`
- Прощания → сигнал выхода
- Сложение чисел → `handle_addition()`
- Запоминание имени → `set_name()`

### 🔹 Обработчики (`handlers.py`)

- **`get_weather(city)`** — отправляет GET-запрос к **Weatherstack API** через библиотеку `requests`. Из JSON-ответа извлекает температуру, скорость ветра и описание погоды на английском. Описание переводится на русский через словарь `WEATHER_TRANSLATIONS`. Возвращает форматированную строку.
- **`handle_weather(match)`** — заглушка для совместимости с паттернами.
- **`handle_greeting()`** — возвращает приветствие.
- **`handle_addition(match)`** — извлекает два числа из регулярного выражения, складывает их и возвращает результат.

### 🔹 Логирование и хранение данных (`logger.py`)

Используется **SQLite** (встроенная БД Python). Создаются две таблицы:
- `logs` — хранит историю диалогов: временная метка, текст пользователя, ответ бота. Перед вставкой проверяется уникальность пары (user_text, bot_text) для избежания дублей.
- `user_info` — хранит имя пользователя в формате «ключ-значение» (ключ `user_name`).

Функции:
- `log_message()` — записывает каждое сообщение и ответ
- `save_user_name()` — сохраняет имя
- `load_user_name()` — загружает имя при запуске

### 🔹 Основной цикл

При запуске создаётся экземпляр `ChatBot`, загружается сохранённое имя. Бот входит в бесконечный цикл, ожидая ввод. На каждое сообщение вызывается `bot.process()`. Если получен сигнал `"EXIT_SIGNAL"` — цикл прерывается. Каждый диалог логируется в базу данных.

### 🔹 Ключевая особенность

В отличие от первой версии, этот бот **отказался от последовательного перебора регулярных выражений** в пользу **триггерного подхода** с извлечением сущностей через **spaCy**. Это позволяет более гибко обрабатывать запросы о погоде, например: «Какая сегодня температура в Москве?», «Дай прогноз погоды», «Что с градусами за окном?» — без жёсткой привязки к шаблону `погода в [город]`.
