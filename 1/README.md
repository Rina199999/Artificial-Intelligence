## Как реализован бот

Бот написан на **Python 3** и построен на основе **регулярных выражений** (модуль `re`). Вся логика разделена на несколько модулей.

### 🔹 Обработка сообщений (`bot.py`)

Класс `ChatBot` получает сообщение от пользователя, приводит его к нижнему регистру и удаляет знаки пунктуации (кроме символа `+`, чтобы не ломать сложение чисел). Затем он последовательно проверяет сообщение по списку паттернов из `patterns.py`. При первом совпадении вызывается соответствующий обработчик. Если ни один паттерн не подошёл — возвращается ответ «Я не понимаю запрос».

### 🔹 Паттерны (`patterns.py`)

Файл возвращает список кортежей: каждый кортеж содержит **регулярное выражение** (объект `re.compile`) и **функцию-обработчик**. Паттерны проверяются в строгом порядке:

- Приветствия (`привет`, `здравствуйте`, `добрый день`) → вызов `handle_greeting()`
- Прощания (`пока`, `до свидания`) → возврат сигнала `"EXIT_SIGNAL"` для выхода из цикла
- Запрос погоды (`погода в Москва`) → вызов `handle_weather()`
- Сложение двух чисел (`5 + 3`) → вызов `handle_addition()`
- Запоминание имени (`меня зовут Алексей`) → вызов метода `set_name()` у экземпляра бота

### 🔹 Обработчики (`handlers.py`)

- **`handle_greeting`** — возвращает приветственную строку.
- **`handle_addition`** — извлекает два числа из совпадения, преобразует их в `float`, складывает и возвращает результат.
- **`handle_weather`** — извлекает название города (если не указано — подставляется `Nizhny Novgorod`). Через библиотеку `requests` отправляет GET-запрос к **Weatherstack API**. Полученный JSON парсится: извлекаются температура, скорость ветра и описание погоды. Описание переводится на русский через словарь `WEATHER_TRANSLATIONS`. Возвращается форматированная строка.

### 🔹 Логирование и хранение данных (`logger.py`)

Используется **SQLite** (встроенная в Python база данных). Создаются две таблицы:

- `logs` — хранит историю диалогов: временная метка, текст пользователя, ответ бота. Перед вставкой проверяется, не было ли уже точно такого же сообщения (чтобы избежать дублей).
- `user_info` — хранит имя пользователя в формате «ключ-значение» (ключ `user_name`).

Функции:
- `log_message()` — записывает каждое сообщение и ответ в `logs`
- `save_user_name()` — сохраняет имя в `user_info`
- `load_user_name()` — загружает имя при запуске бота

### 🔹 Основной цикл (`if __name__ == "__main__"`)

При запуске создаётся экземпляр `ChatBot`, загружается сохранённое имя пользователя. Бот входит в бесконечный цикл, ожидая ввод от пользователя. На каждое сообщение вызывается `bot.process()`. Если получен сигнал `"EXIT_SIGNAL"` — цикл прерывается и бот завершает работу с прощальным сообщением.
