## Как реализован бот

Бот написан на **Python 3** и построен на **гибридной архитектуре**, объединяющей **машинное обучение** (классификация интентов), **регулярные выражения**, **обработку естественного языка через spaCy** и **конечный автомат** для управления диалогом.

### 🔹 Классификация интентов (ML-модель)

**Обучение модели (`train_model.py`):**
- Загружается датасет `dataset.csv` с парами «текст → интент»
- Текст обрабатывается через **spaCy**: приводятся к нижнему регистру, удаляются стоп-слова и пунктуация, выполняется **лемматизация** (приведение слов к нормальной форме)
- Строится пайплайн: **TfidfVectorizer** (преобразование текста в числовые признаки) → **LogisticRegression** (классификатор)
- Готовая модель сохраняется в файл `bot_model.joblib` через библиотеку **joblib**

**Предсказание интента (`bot.py`):**
- Метод `preprocess()` — лемматизация и очистка текста (аналогично обучению)
- Метод `predict_intent()` — применяет модель к тексту, возвращает интент, если уверенность > 0.4, иначе `"unknown"`
- Поддерживаемые интенты: `greeting`, `weather`, `goodbye`, `thanks` (из датасета)

### 🔹 Управление состоянием диалога (DialogState)

Два состояния:
- `START` — обычный режим ожидания команд
- `WAIT_CITY` — режим ожидания названия города

При запуске бота состояние загружается из базы данных (`logger.load_state()`), что позволяет **восстанавливать диалог после перезапуска**.

### 🔹 Обработка сообщений (`bot.process()`)

**Приоритетная схема обработки:**

1. **Проверка состояния** — если бот в `WAIT_CITY`, любое сообщение считается названием города → запрос погоды → сброс состояния в `START`

2. **Классификация интента** (если не в режиме ожидания):
   - `weather` — извлечение города через `extract_city()` (spaCy NER)
     - Если город найден → запрос погоды
     - Если не найден → переход в `WAIT_CITY` + вопрос о городе
   - `greeting` → приветственное сообщение
   - `goodbye` → прощальное сообщение
   - `unknown` и остальные → переход к регулярным выражениям

3. **Регулярные выражения (`patterns.py`)** — резервный механизм:
   - Сложение чисел (`5 + 3`)
   - Сохранение имени (`меня зовут ...`)
   - Приветствия (дублирование на случай низкой уверенности ML)

4. **Ответ по умолчанию** — если ничего не сработало

### 🔹 Извлечение города (spaCy NER)

Метод `extract_city()` использует модель **`ru_core_news_sm`** для поиска именованных сущностей типов:
- **GPE** (Geopolitical Entity) — страны, города, регионы
- **LOC** (Location) — географические локации

При нахождении возвращает лемму сущности с заглавной буквы.

### 🔹 Обработчики (`handlers.py`)

- **`get_weather(city)`** — запрос к **Weatherstack API** через `requests`. Извлекает температуру, скорость ветра, описание погоды. Описание переводится на русский через словарь `WEATHER_TRANSLATIONS`.
- **`handle_addition()`** — сложение двух чисел из регулярного выражения
- **`handle_greeting()`** — приветствие (для паттернов)

### 🔹 Логирование и хранение (`logger.py`)

**SQLite** с таблицами:
- `logs` — временная метка, текст пользователя, ответ бота, **состояние бота** на момент ответа
- `user_info` — хранение ключей:
  - `user_name` — имя пользователя
  - `dialog_state` — текущее состояние диалога (сохраняется между сессиями)

Функции: `log_message()`, `save_user_name()`, `load_user_name()`, `save_state()`, `load_state()`

### 🔹 Основной цикл

1. Загрузка имени пользователя и состояния из БД
2. Загрузка ML-модели (`bot_model.joblib`)
3. Бесконечный цикл ввода
4. Выход по словам `пока` или `выход`
5. Каждый диалог логируется с состоянием бота

### 🔹 Ключевые особенности

1. **Машинное обучение** — бот понимает множество формулировок одного запроса благодаря классификации интентов, а не жёстким паттернам. Например, «сколько градусов на улице», «будет ли дождь», «на улице холодно» — всё распознаётся как `weather`.

2. **Гибкость ML + точность regexp** — ML обрабатывает естественные фразы, регулярные выражения служат резервом для точных команд (математика, установка имени).

3. **Сохранение состояния между сессиями** — состояние диалога (`START` / `WAIT_CITY`) сохраняется в БД, поэтому после перезапуска бот помнит, что пользователь ожидает ответа о городе.

4. **Падение уверенности** — если модель не уверена в предсказании (вероятность < 0.4), возвращается `unknown`, и бот переключается на регулярные выражения или стандартный ответ.

5. **Лемматизация** — приведение слов к нормальной форме (`иду → идти`, `солнечная → солнечный`) улучшает обобщающую способность модели.
