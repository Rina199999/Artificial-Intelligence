## Как реализован бот

Бот написан на **Python 3** и построен на **гибридной архитектуре**, объединяющей **BERT (трансформеры)** для классификации интентов, **spaCy** для извлечения сущностей, **регулярные выражения** для точных команд и **конечный автомат** для управления диалогом.

### 🔹 Классификация интентов на BERT

**Обучение модели (`train_bert.py` / `train_model.py`):**
- Используется предобученная русскоязычная модель **DeepPavlov/rubert-base-cased** от библиотеки **Transformers (Hugging Face)**
- Датасет `dataset.csv` содержит тексты и соответствующие интенты: `greeting`, `weather`, `goodbye`, `thanks`, `time`, `date`, `help`, `smalltalk`, `math`
- Токенизация через `AutoTokenizer` с максимальной длиной 128 токенов
- Обучение на 5-10 эпох с использованием `Trainer` из Hugging Face
- Сохраняются: модель, токенизатор и `label_map.json` (маппинг индексов в названия интентов)

**Предсказание интента (`bot.py`):**
- Загружается сохранённая BERT-модель и токенизатор из папки `intent_model`
- Текст пользователя токенизируется и подаётся в модель
- Выходные логиты преобразуются в вероятности через **softmax**
- Если уверенность > 0.2 — возвращается интент, иначе `"unknown"`
- Для отладки выводится предсказанный интент и уверенность

### 🔹 Управление состоянием диалога (DialogState)

Два состояния:
- `START` — обычный режим ожидания команд
- `WAIT_CITY` — режим ожидания названия города

Состояние загружается из базы данных при запуске и сохраняется при каждом изменении.

### 🔹 Обработка сообщений (`bot.process()`)

**Приоритетная схема обработки:**

1. **Регулярные выражения (`patterns.py`)** — имеют **высший приоритет**:
   - Приветствия → `handle_greeting()`
   - Прощания → `EXIT_SIGNAL`
   - Сложение чисел → `handle_addition()`
   - Сохранение имени → `set_name()`

2. **Проверка состояния** — если бот в `WAIT_CITY`:
   - Извлечение города через `extract_city()` (spaCy NER)
   - Если город не найден — используется всё сообщение как название города
   - Сброс состояния в `START` → запрос погоды

3. **BERT-классификация** (если паттерны не сработали):
   - `weather` → извлечение города → запрос погоды или переход в `WAIT_CITY`
   - `time` → возврат текущего времени
   - `date` → возврат текущей даты
   - `help` → список возможностей бота
   - `smalltalk` → ответ на светскую беседу
   - `greeting` → приветствие с именем пользователя
   - `thanks` → благодарность
   - `unknown` → ответ по умолчанию

### 🔹 Извлечение города (spaCy NER)

Метод `extract_city()` использует модель **`ru_core_news_md`** для поиска именованных сущностей типов:
- **GPE** (Geopolitical Entity) — страны, города, регионы
- **LOC** (Location) — географические локации

Возвращает **лемму** найденной сущности (нормальную форму).

### 🔹 Обработчики (`handlers.py`)

| Функция | Назначение |
|---------|-------------|
| `get_weather(city)` | Запрос к **Weatherstack API** через `requests`. Извлекает температуру, скорость ветра, описание погоды с переводом на русский |
| `get_time_or_date(mode)` | Возвращает текущее время или дату через `datetime.now()` |
| `get_help()` | Возвращает список возможностей бота |
| `handle_smalltalk()` | Ответ на светскую беседу |
| `handle_thanks()` | Ответ на благодарность |
| `handle_addition()` | Сложение двух чисел из регулярного выражения |
| `handle_greeting()` | Приветствие (для паттернов) |

### 🔹 Логирование и хранение (`logger.py`)

**SQLite** с таблицами:
- `logs` — временная метка, текст пользователя, ответ бота, состояние бота
- `user_info` — хранение ключей:
  - `user_name` — имя пользователя
  - `dialog_state` — текущее состояние диалога

### 🔹 Основной цикл

1. Загрузка имени пользователя и состояния из БД
2. Загрузка BERT-модели из папки `intent_model`
3. Бесконечный цикл ввода
4. При получении `EXIT_SIGNAL` — выход с прощанием
5. Каждый диалог логируется в БД


### 🔹 Поддерживаемые интенты

| Интент | Описание | Примеры |
|--------|----------|---------|
| `greeting` | Приветствия | «привет», «здравствуйте» |
| `weather` | Запрос погоды | «какая погода», «сколько градусов» |
| `goodbye` | Прощания | «пока», «до свидания» |
| `thanks` | Благодарности | «спасибо», «благодарю» |
| `time` | Текущее время | «который час», «сколько времени» |
| `date` | Текущая дата | «какое сегодня число», «сегодняшняя дата» |
| `help` | Помощь/справка | «что ты умеешь», «помощь» |
| `smalltalk` | Светская беседа | «как дела», «ты кто» |
| `math` | Математика (запасной) | «посчитай», «реши пример» |

### 🔹 Ключевые особенности

1. **BERT (трансформеры)** — модель учитывает контекст слова в предложении, что даёт значительно лучшее понимание сложных фраз по сравнению с TF-IDF или эмбеддингами.

2. **Низкий порог уверенности (0.2)** — модель чаще будет возвращать интент, снижая количество переходов в `unknown` и делая бота более отзывчивым.

3. **Расширенный набор интентов** — добавлены `time`, `date`, `help`, `smalltalk`, `thanks`, что делает бота полезнее в повседневном общении.

4. **Высший приоритет регулярных выражений** — точные команды (математика, установка имени) обрабатываются до BERT, что гарантирует правильное выполнение даже при низкой уверенности ML.

5. **Персонализация** — приветствие использует сохранённое имя пользователя: `Привет, {name}!`
