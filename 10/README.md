## Как реализован бот

Бот написан на **Python 3** и построен на **гибридной архитектуре**, объединяющей **голосовой ввод через Whisper (STT)**, **синтез речи через VITS (TTS)**, **BERT (трансформеры)** для классификации интентов, **spaCy** для извлечения сущностей, **регулярные выражения** и **конечный автомат** для управления диалогом.

### 🔹 Голосовой ввод (STT) — `voice.py`

**Запись с микрофона:**
- Функция `record_audio_pyaudio()` использует библиотеку **PyAudio**
- Записывается 4 секунды аудио с частотой дискретизации 16000 Гц
- Аудио сохраняется во временный WAV-файл

**Распознавание речи:**
- Используется модель **Whisper (`base`)** от OpenAI
- Функция `load_audio_direct()` читает WAV-файл через `scipy.io.wavfile.read()`
- Аудиоданные нормализуются в диапазон float32
- Whisper транскрибирует аудио на русском языке (`language="ru"`)
- Распознанный текст очищается от лишних символов и лишних пробелов

### 🔹 Синтез речи (TTS) — `voice.py`

**Класс `RussianTTS`:**
- Использует модель **`joefox/tts_vits_ru_hf`** (VITS — Conditional Variational Autoencoder with Adversarial Learning)
- Загружает `VitsModel` и `AutoTokenizer` из Hugging Face

**Нормализация текста перед озвучиванием:**
- Преобразование единиц измерения: `°C` → `градусов Цельсия`, `км/ч` → `километров в час`
- **Преобразование чисел в слова** — c поддержкой:
  - Отдельных цифр (0-9)
  - Чисел 10-20
  - Десятков (20-90)
  - Сотен (100-900)
  - Тысяч (через рекурсивный вызов)
- **Обработка дробных чисел** — `1.5` → `один целых пять`
- **Обработка диапазонов** — `10-15` → `с десяти до пятнадцати`
- Склонение числительных в зависимости от контекста

**Озвучивание:**
- Нормализованный текст токенизируется с параметром `speaker_id`
- Генерация аудиоволны через `model(**inputs).waveform`
- Сохранение в WAV-файл через `soundfile`
- Воспроизведение через **PyAudio** (метод `_play_audio_pyaudio`)

### 🔹 Классификация интентов на BERT

**Обучение модели (`train_bert.py` / `train_model.py`):**
- Используется предобученная русскоязычная модель **DeepPavlov/rubert-base-cased**
- Токенизация через `AutoTokenizer` с максимальной длиной 128 токенов
- Обучение на 5-10 эпох с использованием `Trainer` из Hugging Face
- Сохраняются: модель, токенизатор и `label_map.json`

**Предсказание интента:**
- Текст пользователя токенизируется и подаётся в модель
- Логиты преобразуются в вероятности через **softmax**
- Если уверенность > 0.2 — возвращается интент, иначе `"unknown"`

### 🔹 Управление состоянием (DialogState)

Два состояния:
- `START` — обычный режим
- `WAIT_CITY` — ожидание названия города (после запроса погоды без указания города)

Состояние сохраняется в SQLite и восстанавливается при запуске.

### 🔹 Обработка сообщений (`bot.process()`)

**Приоритетная схема:**

1. **Регулярные выражения** — высший приоритет:
   - Приветствия, прощания, сложение чисел, сохранение имени

2. **Проверка состояния** — если `WAIT_CITY`, извлечение города (или всё сообщение) → запрос погоды

3. **BERT-классификация**:
   - `weather` → извлечение города → погода или `WAIT_CITY`
   - `time` / `date` → текущее время/дата
   - `help` → справка
   - `smalltalk` → светская беседа
   - `greeting` → приветствие с именем
   - `thanks` → благодарность
   - `goodbye` → сигнал выхода

### 🔹 Извлечение города (spaCy NER)

Метод `extract_city()` использует модель **`ru_core_news_md`** для поиска сущностей:
- **GPE** (Geopolitical Entity) — страны, города, регионы
- **LOC** (Location) — географические локации
Возвращает **лемму** найденной сущности.

### 🔹 Обработчики (`handlers.py`)

| Функция | Назначение |
|---------|-------------|
| `get_weather(city)` | Запрос к Weatherstack API (API‑ключ указан в коде), перевод погоды |
| `get_time_or_date(mode)` | Текущее время/дата |
| `get_help()` | Справка |
| `handle_smalltalk()` | Ответ на «как дела» |
| `handle_thanks()` | Ответ на «спасибо» |
| `handle_addition()` | Сложение чисел |

### 🔹 Логирование (`logger.py`)

**SQLite** с таблицами:
- `logs` — timestamp, текст пользователя, ответ бота, состояние
- `user_info` — `user_name`, `dialog_state`

### 🔹 Основной цикл

1. Загрузка имени, состояния и BERT-модели
2. Загрузка моделей TTS (VITS) и STT (Whisper)
3. Бесконечный цикл:
   - Захват голоса через `voice.listen()`
   - Распознавание через Whisper в текст
   - Обработка через `bot.process()`
   - Вывод в консоль и озвучивание через `voice.speak()`
   - Логирование
4. Выход по голосовой команде «пока» или `Ctrl+C`

### 🔹 Ключевые особенности

1. **Полный голосовой интерфейс** — пользователь говорит в микрофон, бот отвечает голосом. Используется современный стек: Whisper для распознавания, VITS для синтеза речи.

2. **Whisper от OpenAI** — высокоточное распознавание русской речи с шумоподавлением и обработкой акцентов.

3. **BERT на CPU** — через `CUDA_VISIBLE_DEVICES="-1"` бот принудительно использует CPU, что обеспечивает совместимость с компьютерами без NVIDIA GPU.

4. **Умная нормализация чисел** — числа и единицы измерения преобразуются в естественно звучащие фразы («15°C» → «пятнадцать градусов Цельсия»).

5. **Обработка ошибок** — бот корректно обрабатывает ситуации, когда речь не распознана, и предлагает повторить попытку.

6. **Сохранение контекста** — состояние диалога сохраняется в БД, бот помнит, что ждёт город, даже после перезапуска.
