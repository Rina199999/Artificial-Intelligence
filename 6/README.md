## Как реализован бот

Бот написан на **Python 3** и построен на **гибридной архитектуре**, объединяющей **векторные представления слов (word embeddings)** через **spaCy**, **логистическую регрессию** для классификации интентов, **регулярные выражения** и **конечный автомат** для управления диалогом.

### 🔹 Классификация интентов на эмбеддингах

**Два подхода к обучению модели:**

1. **TF-IDF + логистическая регрессия (`train_model.py`)**:
   - Текст лемматизируется, очищается от стоп-слов и пунктуации через **spaCy** (`ru_core_news_sm`)
   - Строится пайплайн: `TfidfVectorizer` → `LogisticRegression`
   - Модель сохраняется в `bot_model.joblib`

2. **Векторные эмбеддинги (`train_embeddings.py`)** — **используется в боте**:
   - Загружается модель **spaCy `ru_core_news_md`** (средняя русская модель с word vectors)
   - Каждый текст преобразуется в **вектор фиксированной размерности** (усреднённые эмбеддинги всех токенов)
   - На этих векторах обучается `LogisticRegression`
   - Модель сохраняется в `bot_embeddings_model.joblib`

**Предсказание интента (`bot.py`):**
- Текст пользователя преобразуется в вектор через `nlp(text).vector`
- Вектор подаётся в модель логистической регрессии
- Если уверенность предсказания > 0.3 — возвращается интент, иначе `"unknown"`
- Для отладки выводится уверенность предсказания в консоль

**Поддерживаемые интенты** (из `dataset.csv`): `greeting`, `weather`, `goodbye`, `thanks`

### 🔹 Управление состоянием диалога (DialogState)

Два состояния:
- `START` — обычный режим ожидания команд
- `WAIT_CITY` — режим ожидания названия города

При запуске бота состояние загружается из базы данных (`logger.load_state()`), что позволяет **восстанавливать диалог после перезапуска**.

### 🔹 Обработка сообщений (`bot.process()`)

**Приоритетная схема:**

1. **Проверка состояния** — если бот в `WAIT_CITY`, любое сообщение считается названием города → запрос погоды → сброс состояния

2. **Классификация интента** (через эмбеддинги):
   - `weather` → извлечение города через `extract_city()` (spaCy NER)
     - Город найден → запрос погоды
     - Город не найден → переход в `WAIT_CITY` + вопрос о городе
   - `greeting` → приветственное сообщение
   - `goodbye` → прощальное сообщение (хотя выход обрабатывается отдельно в основном цикле)
   - `unknown` → переход к регулярным выражениям

3. **Регулярные выражения (`patterns.py`)** — резервный механизм:
   - Сложение чисел (`5 + 3`)
   - Сохранение имени (`меня зовут ...`)
   - Приветствия и прощания (на случай низкой уверенности ML)

4. **Ответ по умолчанию** — если ничего не сработало

### 🔹 Извлечение города (spaCy NER)

Метод `extract_city()` использует модель **`ru_core_news_md`** для поиска именованных сущностей типов:
- **GPE** (Geopolitical Entity) — страны, города, регионы
- **LOC** (Location) — географические локации

Возвращает найденное название города без изменений (в отличие от предыдущей версии, где использовалась лемма).

### 🔹 Обработчики (`handlers.py`)

- **`get_weather(city)`** — запрос к **Weatherstack API** через `requests`. Извлекает температуру, скорость ветра, описание погоды. Описание переводится на русский через словарь `WEATHER_TRANSLATIONS`.
- **`handle_addition()`** — сложение двух чисел из регулярного выражения
- **`handle_greeting()`** — приветствие (для паттернов)

### 🔹 Логирование и хранение (`logger.py`)

**SQLite** с таблицами:
- `logs` — временная метка, текст пользователя, ответ бота, состояние бота
- `user_info` — хранение ключей:
  - `user_name` — имя пользователя
  - `dialog_state` — текущее состояние диалога (сохраняется между сессиями)

Функции: `log_message()`, `save_user_name()`, `load_user_name()`, `save_state()`, `load_state()`

### 🔹 Основной цикл

1. Загрузка имени пользователя и состояния из БД
2. Загрузка ML-модели (`bot_embeddings_model.joblib`)
3. Бесконечный цикл ввода
4. Выход по словам `пока` или `выход`
5. Каждый диалог логируется с состоянием бота

### 🔹 Ключевые особенности

1. **Векторные эмбеддинги вместо TF-IDF** — бот использует предобученные русские word vectors из spaCy (`ru_core_news_md`). Это позволяет модели лучше понимать семантическое сходство фраз, даже если они не содержат общих слов.

2. **Размерность векторов** — spaCy `md` модель имеет векторы размерностью 300 (в отличие от `sm`, где векторы отсутствуют). Это даёт более качественное представление смысла текста.

3. **Отладочный вывод** — при каждом предсказании выводится интент и уверенность модели, что упрощает диагностику и дообучение.

4. **Два способа обучения** — в проекте представлены два скрипта обучения (`train_model.py` и `train_embeddings.py`), позволяющие сравнить подходы TF-IDF и эмбеддингов.

5. **Расширенный датасет** — в `dataset.csv` добавлены новые фразы для интента `weather` (например, «мне очень холодно», «зябко на улице»), что улучшает распознавание жалоб на погоду.

6. **Низкий порог уверенности** (0.3) — модель чаще будет возвращать интент, даже при невысокой уверенности, снижая количество переходов в `unknown`.
